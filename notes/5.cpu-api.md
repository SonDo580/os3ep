- Each process has a **process ID (PID)**

# System calls

- `fork()`:
  - used to create new process; creator is called **parent**, new process is called **child**
  - child is a copy of parent, but not exact; it has its own address space and registers
- `wait()`:
  - allow a parent to wait for its child to complete
  - `waitpid()`
- `exec()` family of system calls:
  - allow a child to break free from its similarity to its parent and execute an entirely new programs.
- `pipe()`:
  - create a 1-way communication channel
  - typically used to connect output of 1 process to input of another
- `kill()`:
  - used to send **signals** to a process (directives to pause, die, ...)
- `signal()`:
  - allow a process to catch and handle signals.

# Shell

- Commonly use `fork()`, `wait()`, `exec()` to launch user commands
- Separation of `fork` and `exec` enables features like **input/output redirection**, **pipes**, ...
- Example:
  - use `fork()` to create 2 children
  - before calling `exec()`, use `pipe()` to connect standard output of 1st child to standard input of 2nd child _(1st child redirects standard output to pipe's write-end, 2nd child redirects standard input to pipe's read-end)_

# User & Superuser

- OS allows multiple users onto the system
- A **user** can only control their own processes
- A **superuser** can control all processes; this role should be assumed infrequently and with caution for security reasons

# Tools

- `ps`: which processes are running
- `top`: how much resources each process is using
