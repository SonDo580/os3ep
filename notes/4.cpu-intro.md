- A **process is** a running program
- A process can be **described by** its state:
  - contents of memory in address pace
  - contents of CPU registers: program counter, stack pointer, ...
  - I/O info: open files, ...
- **Process API** consists of calls programs can make related to processes: create, destroy, ...
- **Process states:** running, ready, blocked, ...
  - different events transition a process from 1 state to another: getting scheduled/de-scheduled, initiating I/O, I/O completion, ...
- **Process list:** 
  - contains information about all processes in the system
  - each entry is found in **process control block (PCB)**, a structure that contains information about a specific process.
