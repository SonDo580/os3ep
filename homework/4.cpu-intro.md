# 1

```bash
./process-run.py -l 5:100,5:100
# 2 processes, each consists of 5 instructions, and chances that each instruction is a CPU instruction is 100%
```

- CPU utilization is 100%

# 2

```bash
./process-run.py -l 4:100,1:0
# 1 process with 4 instructions (all to use the CPU),
# 1 that issues an I/O and waits for it to be done (1 IO needs 5 time units, plus 2 CPU actions to when init and complete)
```

- Time needed: 4 + (2 + 5) = 11

# 3

```bash
./process-run.py -l 1:0,4:100
# same as #2 but switch order of processes
```

- Time is reduced due to overlapping: process 1 runs while process 0 is blocked waiting for IO to complete.

```
Time    PID:0           PID:1       CPU     IOs
1       RUN:io          READY       1
2       BLOCKED         RUN:cpu     1       1
3       BLOCKED         RUN:cpu     1       1
4       BLOCKED         RUN:cpu     1       1
5       BLOCKED         RUN:cpu     1       1
6       BLOCKED         DONE                1
7       RUN:io_done     DONE        1
```

# 4

```bash
./process-run.py -l 1:0,4:100 -S SWITCH_ON_END
# same as #3 but system will not switch to another process while one is doing I/O (wait until the process is completely finished)
```

- Time needed is 11 (same as #2), since there's no overlapping

# 5

```bash
./process-run.py -l 1:0,4:100 -S SWITCH_ON_IO
# switch to another process while one is waiting for I/O
```

- Time is 7 (same as #3)

# 6

```bash
./process-run.py -l 3:0,5:100,5:100,5:100 -S SWITCH_ON_IO -I IO_RUN_LATER
# when an I/O completes, the process that issued it is not necessarily run right away; what is running keeps running
```

- While process 0 waiting for the first I/O to complete, all 15 CPU instructions of 3 other processes are executed (the I/O is done in 5 time units, but process 0 must wait).
- The rest of the time is for executing and waiting for I/O of process 0. The CPU is idle (only executes RUN:io and RUN:io_done).
- **Conclusion:** System resources are not utilized effectively.

# 7


```bash
./process-run.py -l 3:0,5:100,5:100,5:100 -S SWITCH_ON_IO -I IO_RUN_IMMEDIATE
# when an I/O completes, immediately run the process that issued it
```

- Each I/O waiting of process 0 is interleaved with executing CPU instructions of other processes. CPU utilization is 100%.
