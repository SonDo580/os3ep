# 4

```bash
./fork.py -A a+b,b+c,c+d,c+e,c-
# ’a’ create ’b’, which in turn creates ’c’, which then creates ’d’ and ’e’. Then, ’c’ exits.
```

- I thought `d` and `e` would also exits, but they didn't.
- Without `-R` flag, they are "attached" to `a`. With `-R` flag, they are attached to `b`.

## Background: Orphaned processes

- An orphaned process is a running computer process whose parent process has terminated or exited.
- The orphaned child continues running and is immediately re-parented, usually to the `init` process.
- `-R` flag of `fork.py` causes re-parenting to local parent.

# 6

```bash
./fork.py -F -t
```

- Predict the series of actions by looking at the final tree

```
Final Process Tree:
a
└── b
    └── c
        └── e

===== Case 1 =====
a forks b
b forks c
c forks e

===== Case 2 =====
a forks ... forks b (...: represents multiple processes in the chain)
b forks c
c forks e
... exits -> b is re-parented to a


===== Case 3 =====
a forks ... (a forks multiple processes)
a forks b
b forks c
c forks e
... exits
(the forks and exits of ... can be interleaved anywhere,
 only fork and exit of each process need to be in order)
```
